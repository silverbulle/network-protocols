# L1 - 物理层 (Physical Layer)

## 职责
在物理介质上传输原始比特流，定义电气、机械、过程和功能特性。

## 核心功能

### 1. 比特编码与信号调制
- 将数据比特转换为物理信号（电、光、无线电波）
- 传统：曼彻斯特、4B/5B、8B/10B
- 现代：64b/66b、128b/130b、PAM-4 + FEC

### 2. 物理介质接口
- 定义连接器的形状、引脚布局
- 定义电气特性（电压、阻抗）
- 定义传输模式（单工/半双工/全双工）

### 3. 传输模式
| 模式 | 说明 | 示例 |
|------|------|------|
| 单工 | 单向传输 | 广播、电视 |
| 半双工 | 双向交替 | 对讲机、传统集线器 |
| 全双工 | 双向同时 | 电话、现代交换机 |

### 4. 拓扑结构
| 拓扑 | 说明 |
|------|------|
| 总线型 | 所有设备共享一条总线 |
| 星型 | 所有设备连接中心节点 |
| 环型 | 设备首尾相连形成环 |
| 网状 | 设备间多路径互联 |
| 树型 | 层次化星型组合 |

## 传输介质

物理介质是 L1 的物理基础，分有线与无线两大类。介质本身决定了速率上限、传输距离和抗干扰能力，上层编码与调制技术都是在既定介质上"压榨"带宽。

## 宽带接入技术（Last Mile / 最后一公里）

接入网（Access Network）是用户到运营商网络之间的物理层链路，决定了家庭/企业的最大带宽上限。主流接入技术按**物理介质**分为三大阵营：铜线（电话线）、同轴电缆、光纤，外加无线接入。

### FTTx 光纤接入架构分类

```
运营商 CO（中心局）                                       用户侧
  ┌──────┐                                            ┌──────────┐
  │ OLT  │─────── 馈线光纤 ──────┬──── 分配光纤 ──────│ 用户终端 │
  └──────┘                       │                    └──────────┘
                                 │
                          ┌──────┴──────┐
                          │ 分光点位置   │
                          │ 决定 FTTx   │
                          └─────────────┘

FTTx 命名 (按光纤到达位置):
  FTTN  (Node)      → 光纤到节点 (路边机柜)    + 铜线入户
  FTTC  (Curb)      → 光纤到路边 (< 300m)      + 铜线入户
  FTTB  (Building)  → 光纤到楼宇 (楼内交换机)  + 双绞线到户
  FTTH  (Home)      → 光纤到户 (ONT 在用户家)  ← 终极方案
  FTTP  (Premises)  → FTTH / FTTB 的统称
  FTTD  (Desk)      → 光纤到桌面（企业场景）
```

### 1. xDSL（铜线电话线接入）

> xDSL = 各类 DSL 的统称，利用现有电话双绞线传输数据，通过频分复用把语音和数据隔离

#### ADSL / ADSL2+（非对称 DSL）
| 项 | 说明 |
|----|------|
| **标准** | ITU-T G.992.1 (ADSL) / G.992.5 (ADSL2+) |
| **调制** | **DMT** (Discrete Multi-Tone, 离散多载波) = OFDM 变体 |
| **子载波** | 256 个（ADSL）/ 512 个（ADSL2+），每个 4.3125 kHz |
| **速率** | 下行 8-24 Mbps / 上行 1-3.5 Mbps（非对称） |
| **距离** | 最长 5 km，速率随距离急剧下降 |
| **语音共存** | POTS 用 0-4 kHz，ADSL 用 26 kHz - 1.1 MHz（通过 POTS Splitter 分离） |

```
频谱划分:
  0 ─────── 4 kHz ────── 26 kHz ──────────────── 1.1 MHz ── 2.2 MHz (ADSL2+)
  │ POTS 语音 │ Guard │ 上行 (US)  │     下行 (DS)      │
  │ (电话)    │       │            │  (子载波多，速率高) │

DMT 原理:
  每个子载波独立 QAM 调制（最高 15 bit/symbol）
  信噪比好的子载波 → 多分配 bit (如 12 bit)
  信噪比差的子载波 → 少分配或不分配 bit
  → 自适应 bit-loading，最大化吞吐量

为什么下行 > 上行？
  - 用户下载多、上传少（典型 Web 行为）
  - 下行子载波数多，上行少（频谱划分不均）
```

#### VDSL / VDSL2（超高速 DSL）
| 项 | 说明 |
|----|------|
| **标准** | ITU-T G.993.2 (VDSL2) |
| **带宽** | 最高 30 MHz（ADSL 仅 2.2 MHz） |
| **速率** | 100 Mbps 对称（短距）/ 下行 200+ Mbps |
| **距离** | 1.5 km 内才有高速，< 300 m 才能满速 |
| **典型部署** | FTTC + VDSL（光纤到路边机柜，最后几百米用铜线） |

#### G.fast（超短距 DSL）
| 项 | 说明 |
|----|------|
| **标准** | ITU-T G.9701 |
| **带宽** | 106 / 212 MHz |
| **速率** | 1 Gbps（< 100 m） |
| **场景** | FTTdp（光纤到分配点），最后 100m 用现有电话线 |

#### xDSL 代际对比

| 技术 | 下行 | 上行 | 最大距离 | 频谱 | 现状 |
|------|------|------|----------|------|------|
| Dial-up (56K) | 56 kbps | 33.6 kbps | 5 km | 0-4 kHz | 已淘汰 |
| ISDN | 128 kbps | 128 kbps | 5 km | 数字 | 已淘汰 |
| ADSL | 8 Mbps | 1 Mbps | 5 km | 1.1 MHz | 遗留 |
| ADSL2+ | 24 Mbps | 3.5 Mbps | 5 km | 2.2 MHz | 遗留 |
| VDSL2 | 100-200 Mbps | 100 Mbps | 1.5 km | 30 MHz | 过渡期 |
| G.fast | 1 Gbps | 1 Gbps | 250 m | 212 MHz | 短距补丁 |

### 2. HFC + DOCSIS（同轴电缆接入）

> **定位**：利用现有有线电视同轴电缆网络做宽带接入的技术。HFC（光纤同轴混合网）把干线改为光纤、最后一段保留同轴；DOCSIS 则定义了在 HFC 上双向传输数据的规范。北美有线电视运营商宽带的主流方案。

> HFC = Hybrid Fiber-Coaxial（光纤同轴混合网），广电有线电视网络改造而来

#### HFC 网络架构
```
运营商 Headend (前端)                        用户家庭
  ┌─────┐                                    ┌────────┐
  │ CMTS│── 光纤 (下行 1550nm 光) ──┐        │ Cable  │
  │     │                            │        │ Modem  │
  └─────┘                            │        │(CM)    │
                                     ▼        └───┬────┘
                               ┌──────────┐       │
                               │ 光节点   │── 同轴电缆 ─┤
                               │ (O/E 转换)│         │
                               └──────────┘         │
                                                    ▼
                                              ┌──────────┐
                                              │ 同轴分配 │
                                              │ (树形拓扑)│
                                              │ 50-75户共享│
                                              └──────────┘

  关键特征:
    - 同轴段是共享介质（类似早期集线器，所有用户共享带宽）
    - 下行广播（所有 CM 都收到，按 MAC 过滤）
    - 上行 TDMA / OFDMA（多 CM 分时/分频发送）
```

#### DOCSIS 标准演进

| 版本 | 下行 | 上行 | 关键技术 | 年代 |
|------|------|------|----------|------|
| **DOCSIS 1.0** | 40 Mbps | 10 Mbps | QAM64/256, TDMA | 1997 |
| **DOCSIS 2.0** | 40 Mbps | 30 Mbps | A-TDMA (抗噪) | 2001 |
| **DOCSIS 3.0** | 1 Gbps | 200 Mbps | **通道捆绑** (最多 32 个 6/8 MHz 通道) | 2006 |
| **DOCSIS 3.1** | **10 Gbps** | 2 Gbps | **OFDM + 4096-QAM**，192 MHz 宽通道 | 2013 |
| **DOCSIS 4.0** | 10 Gbps | **6 Gbps** | FDX (全双工) / ESD (扩展频谱到 1.8 GHz) | 2017 |

#### DOCSIS 关键机制
```
下行 (CMTS → CM):
  广播 MPEG-TS / OFDM 通道（每个 6/8 MHz）
  每个通道 256-QAM ≈ 40-50 Mbps
  DOCSIS 3.0 捆绑 32 通道 = 1.2+ Gbps
  DOCSIS 3.1 用 OFDM (4096-QAM) ≈ 单通道 ~200 Mbps

上行 (CM → CMTS):
  DOCSIS 1.0/2.0: TDMA (分时发送，冲突/噪声敏感)
  DOCSIS 3.0: 通道捆绑 (最多 8 个上行通道)
  DOCSIS 3.1: OFDMA (多用户并行，频谱效率高)
  DOCSIS 4.0 FDX: 同频段双向，需要干扰消除

频段划分 (DOCSIS 3.1):
  5-85 MHz       → 上行
  85-1002 MHz    → 下行
  1002-1218 MHz  → 扩展下行 (可选)
  DOCSIS 4.0 ESD: 扩展到 1.8 GHz
```

### 3. PON（无源光网络，光纤接入）

> **定位**：光纤到户（FTTH）的主流接入技术。用无源分光器把一根局端光纤分给多户共享，局端到用户之间无任何有源设备，低功耗、低故障、易维护。中国三大运营商家庭宽带的主流承载方式（GPON/XGS-PON）。

> PON = Passive Optical Network，从 OLT 到 ONU 之间**无有源设备**（只有分光器），低功耗、低维护

#### PON 网络架构
```
运营商 CO                                    用户侧
  ┌─────┐                                    ┌──────┐
  │ OLT │── 馈线光纤 ──┐                     │ ONT  │── 家庭 LAN
  │(局端)│              │                     │(光猫)│
  └─────┘              ▼                     └──────┘
                 ┌──────────┐                   ▲
                 │ 无源分光器 │                   │
                 │ (1:32/64) │                   │
                 └─────┬────┘                   │
                       │── 分配光纤 ─────────────┘
                       │
                 ┌─────┴─────┐
                 │   分光器   │
                 ├───────────┤
                 │  ONT/ONU  │  (每根光纤接 32-64 户)
                 │  ONT/ONU  │
                 │  ONT/ONU  │
                 └───────────┘

  关键:
    - 分光器 = 光功率 N 等分（1:32 每路损耗 ~15 dB）
    - 下行广播（OLT 发给所有 ONT，ONT 按 LLID/GEM-Port 过滤）
    - 上行 TDMA（ONT 在授权时隙发送，避免冲突）
    - 无源 = 不需要供电/散热，故障率低
```

#### PON 标准演进

| 标准 | 组织 | 下行 | 上行 | 波长 | 分光比 |
|------|------|------|------|------|--------|
| **APON/BPON** | ITU-T G.983 | 155/622 Mbps | 155/622 Mbps | 1490/1310 nm | 1:32 |
| **EPON** | IEEE 802.3ah | 1.25 Gbps | 1.25 Gbps | 1490/1310 nm | 1:32 |
| **GPON** | ITU-T G.984 | **2.5 Gbps** | 1.25 Gbps | 1490/1310 nm | 1:64 |
| **10G-EPON** | IEEE 802.3av | 10 Gbps | 10/1 Gbps | 1577/1270 nm | 1:128 |
| **XG-PON** | ITU-T G.987 | 10 Gbps | 2.5 Gbps | 1577/1270 nm | 1:128 |
| **XGS-PON** | ITU-T G.9807 | 10 Gbps | **10 Gbps** | 1577/1270 nm | 1:128 |
| **NG-PON2** | ITU-T G.989 | 40 Gbps | 40 Gbps | **TWDM 多波长** | 1:128 |
| **50G-PON** | ITU-T G.9804 | 50 Gbps | 12.5-50 G | — | 1:128 |

> **中国三大运营商现状**：
> - 中国电信 / 联通：GPON → XG-PON / XGS-PON
> - 中国移动：早期 EPON，现转 GPON / XG-PON
> - 千兆宽带主流：XGS-PON (10G 对称)

#### GPON 关键技术

```
帧结构 (GTC 帧, 125 µs 周期):
  ┌──────────┬───────────────────────────────┐
  │ PCBd     │ Payload (GEM Frames)          │
  │ (物理层  │ 多个 GEM Frame 拼接           │
  │  控制块) │ 每帧: GEM Header + Payload    │
  └──────────┴───────────────────────────────┘

  PCBd 包含:
    - US BW Map: 上行带宽分配（告诉每个 ONT 什么时候发）
    - DBRu: 上行报告
    - Ploam: 物理层 OAM

下行 (广播):
  OLT 广播 GTC 帧 → 所有 ONT 收到
  每个 ONT 按 GEM Port ID 过滤出自己的数据
  安全: 下行 AES-128 加密（防窃听）

上行 (TDMA):
  OLT 通过 BW Map 分配时隙给每个 ONT
  ONT 仅在授权时隙发送 → 避免光信号冲突
  OLT 做测距 (Ranging) 补偿不同 ONT 的光距离差

DBA (动态带宽分配):
  OLT 实时统计各 ONT 队列长度
  按 T-CONT 类型分配:
    - Fixed: 固定带宽（语音）
    - Assured: 保证带宽（视频）
    - Non-Assured: 尽力而为（上网）
    - Best Effort: 空闲才分配
```

#### WDM-PON / TWDM-PON（下一代）
```
传统 PON: 单波长承载所有用户，速率受限
WDM-PON: 每用户 (或每 N 用户) 独立波长
  优点: 带宽独占，安全，扩展性好
  缺点: OLT 侧需可调激光器，成本高

TWDM-PON (NG-PON2):
  4-8 个波长 × 10 Gbps = 40-80 Gbps
  每个波长可独立跑 XG-PON / XGS-PON
  波长可动态分配给不同 ONT
```

### 4. 其他接入技术

#### 以太网专线 (Ethernet over Fiber, EoF)
> **定位**：运营商光纤直连用户楼宇、以标准以太网承载的专线接入。带宽独占、SLA 可保障，是企业总部、数据中心互联、金融专线等对时延和可用性要求高场景的主流方案。

- **企业接入主流**：运营商光纤直连用户楼宇，L2 以太网专线
- 速率：10 Mbps / 100 Mbps / 1 Gbps / 10 Gbps 可定制
- SLA 保证（带宽、时延、可用性 99.9%+）
- 价格高，典型用于：企业总部、数据中心互联、金融专线

#### 无线固定接入 (FWA, Fixed Wireless Access)
> **定位**：用无线（主要是 5G）替代有线入户的固定宽带接入方式，用户侧用 CPE 把蜂窝信号转为家庭 Wi-Fi。适合铺光纤成本过高的农村、偏远地区或临时场所。

- **5G FWA**：运营商 5G CPE 替代光纤入户（农村/偏远地区）
- 下行 100-500 Mbps，上行 30-100 Mbps
- 典型：美国 T-Mobile Home Internet、Verizon 5G Home
- 中国：部分农村地区 5G CPE 替代 FTTH

#### 卫星接入
> **定位**：用卫星做接入链路的宽带方式，覆盖光纤无法到达的偏远地区、海上和航空场景。低轨（LEO）星座（如 Starlink）大幅降低了延迟，使其从"最后选择"变为可用的主流偏远接入方案。

| 类型 | 代表 | 延迟 | 速率 | 场景 |
|------|------|------|------|------|
| GEO（同步） | 传统卫星 | 600 ms+ | 几十 Mbps | 遗留，体验差 |
| **LEO（低轨）** | **Starlink** | 20-40 ms | 100-300 Mbps | 农村/海上/航空 |
| MEO | O3b (SES) | 120-150 ms | Gbps 级 | 运营商回传 |

```
Starlink (SpaceX):
  5000+ 颗 LEO 卫星 (550 km 轨道)
  用户终端: 相控阵天线 (自动跟踪卫星)
  星间链路: 激光通信 (卫星间直连，减少地面站依赖)
  应用场景: 偏远地区、海上、飞机 Wi-Fi
```

#### 电力线通信 (PLC, Power Line Communication)
> **定位**：复用现有电力线传输数据的接入/通信方式，无需重新布线。主要用于家庭内部联网（HomePlug）和电力公司的智能电表回传（AMI），缺点是电力线干扰大、速率不稳定。

- 利用电力线传输数据
- HomePlug AV2：~500 Mbps（家庭内部）
- 场景：智能家居、电力公司 AMI（智能电表）
- 缺点：干扰大，速率不稳定

#### 遗留技术（已基本淘汰）

| 技术 | 速率 | 介质 | 现状 |
|------|------|------|------|
| **Dial-up 56K** | 56 kbps | 电话线 + Modem | 已淘汰 |
| **ISDN** | 64/128 kbps | 电话线（数字） | 已淘汰 |
| **T1/E1 专线** | 1.544 / 2.048 Mbps | 双绞线 | 部分遗留企业 |
| **Frame Relay** | 56 kbps - 45 Mbps | WAN 虚电路 | 被 MPLS 取代 |

```

```

| **XGS-PON** | 10 G | 10 G | 光纤 | 共享 (1:128) | 3-10 ms | 千兆宽带 |
| **FTTH 点对点** | 1-10 G | 1-10 G | 光纤 | 独占 | 1-5 ms | 企业 |
| **5G FWA** | 500 M | 100 M | 无线 | 共享 (小区) | 10-30 ms | 农村/临时 |
| **Starlink** | 300 M | 30 M | LEO 卫星 | 共享 | 20-40 ms | 偏远/海上 |

### 7. 接入网与上层协议的关系

```
家庭用户访问互联网，L1 接入网之上的协议栈:

用户设备           光猫 (ONT)              BRAS/BNG              互联网
┌──────┐         ┌─────────┐             ┌─────────┐           ┌──────┐
│ PC   │── ETH ──│ ONT     │── PON/GPON ─│ BRAS    │── IP/MPLS─│ 服务器│
│      │         │ (L1/L2) │             │ (PPPoE/ │           │      │
│      │         │         │             │  IPoE)  │           │      │
└──────┘         └─────────┘             └─────────┘           └──────┘

家庭宽带典型协议栈:
  用户 PC
    ↓ Ethernet (L2)
  家用路由器 (NAT + PPPoE Client)
    ↓ PPPoE (L2) 或 IPoE
  光猫 ONT (L1 转换: ETH ↔ PON)
    ↓ GPON / XGS-PON (L1 接入网)
  OLT (L2/L3)
    ↓ IP/MPLS
  BRAS/BNG (宽带接入服务器，做 PPPoE 终结、QoS、计费)
    ↓ IP/MPLS
  运营商核心网
    ↓ IP
  互联网

企业专线典型协议栈:
  企业 LAN
    ↓ Ethernet (VLAN 标记)
  CPE 路由器
    ↓ Ethernet 专线 (EoF) / MPLS VPN
  运营商 PE 路由器
    ↓ IP/MPLS
  运营商核心网
```

## 常见编码方式

物理层编码要解决三个核心问题：
1. **时钟恢复**：保证足够的信号跳变，让接收方提取时钟
2. **直流平衡**：长时间不跳变会造成基线漂移
3. **效率**：编码开销越小，有效带宽越高

### 传统编码

#### 曼彻斯特编码（10BASE-T 以太网）
> **定位**：最早的实用线路编码，通过每个比特中间的一次电平跳变同时携带数据和时钟，让接收方无需独立时钟线就能同步。代价是效率仅 50%，只用于低速经典以太网。

```
数据:     1    0    1    1    0
信号:   ┌─┐  ┌──┐┌─┐ ┌─┐ ┌──┐
        │ │  │  ││ │ │ │ │  │
        └─┘  └──┘└─┘ └─┘ └──┘
        高→低=1  低→高=0
```
- 每个比特都有跳变，时钟恢复完美
- 效率仅 **50%**（2 符号才传 1 bit），不适合高速

#### 4B/5B（100BASE-TX / FDDI）
> **定位**：分组型线路编码，把 4 bit 数据查表映射为 5 bit 码字，靠冗余位保证足够跳变用于时钟恢复。相比曼彻斯特效率翻倍（80%），是百兆以太网的过渡方案。

- 每 4 bit 数据映射为 5 bit 码字
- 选取的码字保证至少 2 个 1，避免长连 0
- 效率：**80%**

#### 8B/10B（千兆以太网 / SATA / USB 3.0 / DisplayPort）
> **定位**：查表型线路编码的集大成者，将 8 bit 映射为 10 bit，严格保证直流平衡并提供 K 字符控制码。在 10GbE 之前是高速串行链路的事实标准，至今仍用于 SATA/USB3/DisplayPort。

- 每 8 bit 数据映射为 10 bit 码字
- 严格 DC 平衡（运行不均衡度控制在 ±1）
- 提供 K 字符（控制码，如 COM、IDLE）
- 效率：**80%**
- 仍是 10GbE 之前最主流的编码

### 64b/66b 编码（10GbE / 25GbE / 100GbE / PCIe Gen6 / USB4）

> IEEE 802.3ae（10GbE）引入，现代高速链路的基石

#### 设计思路
抛弃 8B/10B 的查表映射，改用 **扰码 + 同步头**：

```
64 bit 数据 ──→ 扰码 (自同步多项式 x⁵⁸+x³⁹+1) ──→ 加 2 bit 同步头 ──→ 66 bit 块

66 bit 块结构:
  ┌─────┬──────────────────────────┐
  │ SH  │       64 bit payload     │
  │ 2 b │        (scrambled)       │
  └─────┴──────────────────────────┘

同步头 (Sync Header):
  01 → data block（payload 是扰码后的数据）
  10 → control block（payload 含 8 bit 类型字段 + 56 bit 数据/控制）
  00/11 → 非法，用于块对齐错误检测
```

#### 关键机制

| 机制 | 说明 |
|------|------|
| **扰码 (Scrambling)** | 伪随机化数据，保证信号统计特性接近白噪声，避免长连 0/1 |
| **同步头** | 每块开头固定 2 bit（01/10），接收方据此做块对齐 |
| **DC 平衡** | 扰码自然达成，无需额外编码位 |
| **效率** | **96.97%**（64/66），远超 8B/10B 的 80% |

#### 为什么能取代 8B/10B？
- 10 Gbps 下，8B/10B 需要 12.5 Gbaud 的符号速率 → 硬件成本高
- 64b/66b 只需 10.3125 Gbaud → 物理可行
- 扰码替代查表，硬件实现更简单（线性反馈移位寄存器 LFSR）

#### 应用
| 标准 | 说明 |
|------|------|
| 10GBASE-R（10GbE） | 首个采用 64b/66b 的以太网标准 |
| 25GBASE-R / 40GBASE-R4 | 25G/40G 以太网 |
| 100GBASE-R（100GbE） | 4 通道 × 25.78 Gbaud |
| PCIe Gen 6 | 沿用 64b/66b，配合 PAM-4 |
| USB4 | 基于 Thunderbolt，使用 64b/66b |
| InfiniBand HDR/NDR | 100G/400G 数据中心互联 |

### 128b/130b（PCIe Gen3 / Gen4 / Gen5）
> **定位**：PCIe 阵营的高速编码，结构与 64b/66b 同源但块更大（128 bit + 2 bit 同步头），效率达 98.46%，满足 PCIe 对带宽的极致追求。以太网未采用此编码。

- PCIe 阵营的选择（非以太网）
- 与 64b/66b 结构类似，但块更大
- 效率：**98.46%**（更高），适合 PCIe 对带宽敏感的场景
- 以太网未采用（已有 64b/66b 生态）

### 256b/257b（100GbE OTN / IEEE 802.3bj）
> **定位**：效率逼近理论极限的编码（99.61%），用于 OTN 光传输网络把多个 64b/66b 块聚合后打包，开销极低但实现复杂，面向超长距光传输场景。

- 效率 **99.61%**，接近理论极限
- 用于 OTN（光传输网络）转发器，把多个 64b/66b 块聚合后打包
- 开销极低，但实现复杂

### PAM-4（25G/50G/100G/400G/800G 以太网）

> **定位**：多电平调制技术，用 4 个电压电平在每个符号携带 2 bit，在相同符号速率下翻倍比特率。是 SerDes 速率逼近物理极限后取代 NRZ 的关键，25G 及以上以太网和 800G 的物理层基础。代价是对噪声更敏感，必须配合 FEC。

> 从 NRZ 到多电平调制的跨越

```
NRZ (PAM-2):  2 电平，每符号 1 bit
                ┌───┐
            1   │   │
                └───┘
            0

PAM-4:        4 电平，每符号 2 bit
                ┌───┐  ← 电平 3 (11)
            3   │   │
                └───┘
                ┌───┐  ← 电平 2 (10)
            2   │   │
                └───┘
                ┌───┐  ← 电平 1 (01)
            1   │   │
                └───┘
                ┌───┐  ← 电平 0 (00)
            0   │   │
                └───┘
```

#### 为什么必须用 PAM-4？
- SerDes 电信号速率到 **~25 Gbaud** 后，铜缆/PCB 的损耗使 NRZ 不可行
- PAM-4 用相同符号速率传输 **2 倍** 比特
- 代价：信噪比需求 ↑ 约 9.5 dB → 必须配合 **FEC（前向纠错）**

#### 应用
| 标准 | 通道组合 |
|------|----------|
| 25GBASE-T | 铜缆 + PAM-4 |
| 50GAUI / 100GAUI-2 | 2 通道 × 50G PAM-4 |
| 400GBASE-DR4 | 4 通道 × 100G PAM-4（光） |
| 800GBASE-R | 8 通道 × 100G PAM-4 |
| PCIe Gen 6 | 配合 64b/66b + FEC |

### FEC（前向纠错）—— 现代高速链路的标配
> **定位**：在物理层用冗余编码实现"在线纠错"——接收端直接纠正比特错误，无需重传。在 PAM-4 等多电平调制导致裸 BER 偏高时，FEC 把 BER 从 10⁻⁶ 降到 10⁻¹²，是高速链路（100G/400G/800G）可用性的保障。代价是增加数 ns 延迟和约 5-7% 带宽开销。

- PAM-4 信噪比低，裸误码率（BER ~10⁻⁶）无法接受
- FEC 在发送端加入冗余码，接收端纠错，将 BER 降到 **10⁻¹² ~ 10⁻¹⁵**
- 代价：增加几 ns 延迟
- 类型：
  - **RS-FEC**（Reed-Solomon）：100G/400G 以太网主流
  - **KP4 FEC**：25GBASE-T 使用
  - **BASE-R FEC**（Clause 74）：轻量级，早期 10G/25G

### 编码方式总览对比

| 编码 | 效率 | 典型应用 | 速率 | 特点 |
|------|------|----------|------|------|
| 曼彻斯特 | 50% | 10BASE-T | 10 Mbps | 经典教学用，已过时 |
| 4B/5B | 80% | 100BASE-TX, FDDI | 100 Mbps | 简单，控制字符少 |
| 8B/10B | 80% | 1GbE, SATA, USB3, DP | 1-6 Gbps | DC 平衡，K 字符丰富 |
| 64b/66b | 96.97% | 10/25/40/100GbE, PCIe6 | 10-64 Gbps | 现代高速标准，扰码 |
| 128b/130b | 98.46% | PCIe Gen3/4/5 | 8-32 GT/s | PCIe 专用 |
| 256b/257b | 99.61% | 100G OTN | 100+ Gbps | 光传输网络 |
| PAM-4 | ×2 bit/symbol | 25G-800G 以太网 | 25-100 Gbaud | 多电平，需配 FEC |

## 常见物理层设备

物理层设备工作在纯比特域，只负责信号的再生、转换、放大，不理解任何帧结构或地址——这是它们与 L2 交换机/L3 路由器的本质区别。

| 设备 | 功能 |
|------|------|
| 中继器 (Repeater) | 放大/再生信号，扩展传输距离 |
| 集线器 (Hub) | 多端口中继器，所有端口共享冲突域 |
| 光纤收发器 | 电信号 ↔ 光信号转换 |
| 调制解调器 (Modem) | 数字 ↔ 模拟信号转换 |

## 物理层关键参数
- **带宽**：信道可传输的最高频率（Hz）
- **数据速率**：实际传输的比特率（bps）
- **衰减**：信号随距离减弱的程度
- **时延**：信号从发送到接收的时间
- **误码率 (BER)**：传输错误的比特比例（裸 ~10⁻⁶，FEC 后 ~10⁻¹²）
- **FEC 纠错**：物理层在线纠错（Reed-Solomon / LDPC / 卷积码），ns 级延迟

> 详见 [protocols/error-control.md L1](../protocols/error-control.md#l1-物理层--比特级纠错fec)
