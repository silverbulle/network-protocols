# L4 - 传输层 (Transport Layer)

## 职责
提供端到端（进程到进程）的数据传输服务，负责数据的分段、流量控制、差错恢复。

## 核心功能

### 1. 分段与重组
- 将上层数据分割为适合网络传输的段（Segment）
- 接收端按序重组

### 2. 端到端连接管理
- 建立、维护、终止逻辑连接
- TCP 三次握手 / 四次挥手

### 3. 流量控制
- 防止发送方发送过快导致接收方缓冲区溢出
- TCP 滑动窗口机制

### 4. 差错控制
- 校验和检测数据完整性
- 确认机制（ACK）+ 超时重传
- 序列号保证有序交付

### 5. 多路复用
- 通过端口号区分不同应用进程
- 多个应用共享同一网络连接

## TCP vs UDP

| 特性 | TCP | UDP |
|------|-----|-----|
| 连接类型 | 面向连接 | 无连接 |
| 可靠性 | 可靠（确认、重传、排序） | 不可靠（尽力交付） |
| 有序性 | 保证有序 | 不保证 |
| 流量控制 | 有（滑动窗口） | 无 |
| 拥塞控制 | 有 | 无 |
| 头部开销 | 20-60 字节 | 8 字节 |
| 速度 | 较慢 | 较快 |
| 适用场景 | Web、文件传输、邮件 | DNS、视频、游戏 |

## 三次握手
```
Client                        Server
  │                              │
  │──── SYN (seq=x) ────────────→│
  │                              │
  │←─── SYN+ACK (seq=y,ack=x+1) │
  │                              │
  │──── ACK (ack=y+1) ──────────→│
  │                              │
  │════ 连接建立 ════════════════│
```

## 四次挥手
```
Client                        Server
  │                              │
  │──── FIN (seq=u) ────────────→│  客户端关闭发送
  │                              │
  │←─── ACK (ack=u+1) ──────────│  服务器确认
  │                              │  (服务器可能还有数据要发)
  │←─── FIN (seq=w) ────────────│  服务器关闭发送
  │                              │
  │──── ACK (ack=w+1) ──────────→│  客户端确认
  │                              │
  │  [TIME_WAIT: 2MSL]          │  等待可能的迟到报文
  │                              │
  │════ 连接关闭 ════════════════│
```
