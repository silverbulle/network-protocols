# L5 - 会话层 (Session Layer)

## 职责
建立、管理和终止两个通信实体之间的会话（Session）。

## 核心功能

### 1. 会话建立与维护
- 建立通信双方的逻辑连接
- 管理会话状态（全双工 / 半双工）
- 会话标识与跟踪

### 2. 同步与检查点
- 在数据流中插入同步点（Sync Points）
- 当传输中断时，可从最近的同步点恢复，无需重传全部数据
- **主同步点**：确认后才继续，保证数据完整
- **次同步点**：可在主同步点内自由设置，用于细粒度恢复

### 3. 对话控制
- **全双工**：双方可同时收发
- **半双工**：交替收发，需要令牌（Token）控制
- **令牌管理**：谁持有令牌谁才能发送

## 相关协议与技术

| 协议/技术 | 说明 |
|-----------|------|
| NetBIOS | Windows 网络中名称解析与会话管理 |
| RPC (Remote Procedure Call) | 远程过程调用，隐含会话管理 |
| PPTP | 点对点隧道协议，管理 VPN 会话 |
| SIP | VoIP 会话初始化协议 |
| NFS | 网络文件系统（会话管理由 RPC 层处理） |
| Socket API | BSD Socket 提供的会话管理接口 |

## 会话恢复示例
```
发送方                          接收方
  │─────── 数据块 1 ────────────→│
  │─────── 数据块 2 ────────────→│
  │─────── [主同步点] ──────────→│  ← 接收方确认
  │─────── 数据块 3 ────────────→│
  │─────── 数据块 4 ────────────→│  ← 传输中断！
  │                               │
  │─────── 重新连接 ────────────→│
  │←─── 从主同步点后恢复 ───────│
  │─────── 数据块 3 (重传) ─────→│
  │─────── 数据块 4 (重传) ─────→│
```

> 在实际 TCP/IP 实现中，会话层功能大多由传输层（TCP 连接管理）和应用层协议共同实现。
