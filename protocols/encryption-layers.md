# 加密在网络模型中的层级分布

## 概述

加密可以在 OSI/TCP-IP 模型的**每一层**实现，不同层的加密方案保护的数据范围、部署位置、性能特征完全不同。核心原则：

> **越底层加密 → 保护范围越大（含头部），但部署越重；越上层加密 → 部署越灵活，但保护范围越小。**

```
┌─────────────────────────────────────────────────────────────┐
│ 应用层 (L7)    TLS/SSL · SSH · PGP/S/MIME · Signal · QUIC   │  端到端，保护应用数据
├─────────────────────────────────────────────────────────────┤
│ 传输层 (L4)    TLS 1.3 · QUIC (UDP 上)                      │  端到端，保护字节流
├─────────────────────────────────────────────────────────────┤
│ 网络层 (L3)    IPsec · WireGuard (UDP 封装)                 │  逐跳或端到端，保护 IP 包
├─────────────────────────────────────────────────────────────┤
│ L2.5           MPLS VPN (无加密，逻辑隔离)                   │  不加密，靠标签隔离
├─────────────────────────────────────────────────────────────┤
│ 链路层 (L2)    MACsec · WPA2/3 (Wi-Fi) · PPP CHAP/EAP       │  单跳，保护帧
├─────────────────────────────────────────────────────────────┤
│ 物理层 (L1)    光纤物理安全 · 扩频/跳频 · 量子密钥分发 QKD  │  物理信号层
└─────────────────────────────────────────────────────────────┘
```

## 各层加密详解

### L7 应用层加密

**保护对象**：应用数据（HTTP body、邮件正文、即时消息等）

| 协议 | 场景 | 特点 |
|------|------|------|
| **PGP / S/MIME** | 邮件端到端加密 | 用户持有密钥，邮件服务器看不到内容 |
| **Signal Protocol** | IM 端到端加密 | Double Ratchet 算法，前向保密 + 后向保密 |
| **OpenPGP / age** | 文件加密 | 离线加密，公钥或对称密钥 |
| **DNS over HTTPS (DoH)** | DNS 查询隐私 | 复用 TLS，隐藏 DNS 流量 |
| **ESNI / ECH** | TLS 中 SNI 字段加密 | 防止中间人看到访问哪个域名 |

**特点**：
- 真正的端到端（E2E）—— 中间设备完全无法解密
- 应用自己控制密钥与策略
- 不保护任何网络层头部（IP、端口、长度可见）

### L4 传输层加密

**保护对象**：TCP/UDP 载荷（应用层数据）+ 部分传输层字段

#### TLS (Transport Layer Security)
| 项 | 说明 |
|----|------|
| **版本** | TLS 1.2 (RFC 5246) / TLS 1.3 (RFC 8446, 主流) |
| **运行位置** | TCP 之上（应用层与传输层之间） |
| **端口** | 443 (HTTPS)、465 (SMTPS)、993 (IMAPS) 等 |
| **加密范围** | TCP 载荷全部加密，TCP 头部 + IP 头部可见 |
| **前向保密** | TLS 1.3 强制（ECDHE / DHE） |
| **握手** | TLS 1.3 仅 1-RTT（支持 0-RTT 重连） |

```
┌──────┬──────┬─────────┬─────────────────────────┐
│ ETH  │ IP   │ TCP 头  │  TLS Record (加密)      │
│ 14B  │ 20B  │ 20B可见 │  ┌────┬──────────────┐  │
│      │      │ 端口可见│  │头5B│ Payload 加密  │  │
│      │      │         │  └────┴──────────────┘  │
└──────┴──────┴─────────┴─────────────────────────┘
      可见         可见           加密
```

#### QUIC (Quick UDP Internet Connections)
| 项 | 说明 |
|----|------|
| **RFC** | 9000 (QUIC v1), 9369 (QUIC v2) |
| **运行位置** | UDP 之上（替代 TCP + TLS） |
| **加密范围** | **几乎整个 QUIC 包头** 都加密（含大部分控制字段） |
| **优势** | 抗中间设备 ossification、0-RTT 连接建立、无队头阻塞 |
| **应用** | HTTP/3、WebRTC (部分)、DNS over QUIC |

```
┌──────┬──────┬───────┬──────────────────────────────┐
│ ETH  │ IP   │ UDP头 │  QUIC Packet (几乎全加密)    │
│      │      │ 可见  │  Header 大部分 + Payload 加密│
└──────┴──────┴───────┴──────────────────────────────┘
```

### L3 网络层加密

**保护对象**：IP 包（含 IP 载荷，可选含 IP 头部）

#### IPsec (Internet Protocol Security)

| 项 | 说明 |
|----|------|
| **RFC** | 4301-4309 (架构/AH/ESP/IKEv2 等) |
| **运行位置** | IP 层（协议号 50=ESP, 51=AH, UDP 500/4500=IKE） |
| **加密范围** | 取决于模式（见下） |
| **算法** | AES-GCM / AES-CBC / ChaCha20-Poly1305 + SHA2/HMAC |
| **密钥协商** | IKEv1 / IKEv2（Diffie-Hellman） |
| **应用** | 站点到站点 VPN、远程访问 VPN、移动 VPN |

**两种协议**：
| 协议 | 功能 | 加密 | 认证 | 现状 |
|------|------|------|------|------|
| **AH** (认证头, 协议号 51) | 完整性 + 源认证 | ❌ 不加密 | ✅ 含 IP 头 | 基本弃用（NAT 不友好） |
| **ESP** (封装安全载荷, 协议号 50) | 加密 + 完整性 + 源认证 | ✅ | ✅（不含外层 IP 头） | 主流 |

**两种模式**：

```
传输模式 (Transport Mode) —— 端到端 (主机 ↔ 主机)
原始包:   [IP 头] [TCP] [Data]
ESP:      [IP 头] [ESP 头] [TCP] [Data(加密)] [ESP 尾] [ESP 认证]
          ↑ 保留原始 IP 头，只加密载荷

隧道模式 (Tunnel Mode) —— 网关 ↔ 网关 (典型 VPN)
原始包:   [IP 头_内] [TCP] [Data]
ESP:      [IP 头_外] [ESP 头] [IP 头_内(加密)] [TCP(加密)] [Data(加密)] [ESP 尾] [ESP 认证]
          ↑ 加上新外层 IP 头，整个原始包被加密
```

**VPN 完整流程**：
```
站点 A 主机                  VPN 网关 A                   VPN 网关 B                 站点 B 主机
    │  [IP_A] [Data]            │  [IP_GA][ESP][IP_A][Data]      │  [IP_A] [Data]          │
    ├───────────────────────► │ ──── 隧道(公网) ─────────────► │ ─────────────────────► │
    │  明文                    │  加密(外层 IP 是网关公网 IP)    │  解密还原                │
```

#### WireGuard

| 项 | 说明 |
|----|------|
| **作者** | Jason Donenfeld (Linux 内核原生，2020 合入) |
| **算法** | 固定组合：ChaCha20 + Poly1305 + Curve25519 + BLAKE2s + SipHash |
| **代码量** | ~4000 行（对比 IPsec/OpenVPN 数十万行） |
| **特点** | 极简、高速、无配置复杂状态机、默认前向保密 |
| **封装** | 跑在 UDP 上（默认端口 51820），隧道模式 |

```
WireGuard 封装:
原始包:   [IP 头_内] [TCP/UDP] [Data]
WireGuard: [IP 头_外] [UDP] [WG 头] [加密的 IP 头_内 + 载荷]
```

#### 对比：IPsec vs WireGuard vs OpenVPN

| 维度 | IPsec | WireGuard | OpenVPN |
|------|-------|-----------|---------|
| 协议栈 | IP 层 | UDP 上 | UDP/TCP 上 (用户空间) |
| 算法灵活性 | 多算法协商 | 固定最优组合 | 多算法 |
| 代码量 | 数十万行 | ~4k 行 | 数万行 |
| NAT 穿透 | 需 NAT-T (UDP 4500) | 原生 UDP | 用户空间易穿透 |
| 性能 | 高（内核 + 硬件加速） | 最高（内核 + ChaCha20） | 中（用户空间加解密） |
| 部署复杂度 | 高（策略/证书/预共享） | 低（公钥交换） | 中 |
| 主流场景 | 企业站点 VPN、运营商 | 个人/小团队、现代部署 | 遗留远程访问 VPN |

### L2.5 标签层

**严格来说不加密**，靠标签做逻辑隔离（运营商信任）：
- **MPLS VPN**：标签隔离 VRF，数据明文传输
- **VXLAN**：UDP 4789 封装，无加密（可叠加 IPsec/MACsec）

若需加密：MPLS + IPsec 隧道 / MPLS + MACsec

### L2 链路层加密

**保护对象**：整个以太网帧（含 L3/L4 载荷，部分方案含 L2 头）

| 协议 | 场景 | 说明 |
|------|------|------|
| **MACsec (802.1AE)** | 数据中心 / 企业有线网 | 逐跳加密，保护整个以太网帧（除前导码/FCS），用 SecTAG + ICV |
| **WPA2-PSK / WPA3-SAE** | Wi-Fi | 无线空口加密，CCMP (AES) 或 GCMP (WPA3) |
| **802.1X + EAP** | 接入认证 | 不是加密协议，但协商出加密密钥（配合 MACsec/WPA） |
| **PPP + CHAP/EAP** | 拨号/PPP 链路 | 认证+可选加密（MPPE） |
| **L2TP** | 隧道（常与 IPsec 叠加） | 本身不加密，L2TP/IPsec = IPsec 隧道包 L2TP 帧 |

#### MACsec 详解
```
┌──────┬───────────┬────────────┬──────┬─────┐
│ DMAC │ SecTAG    │ Payload    │ ICV  │ FCS │
│ SMAC │ (16B)     │ (加密)     │ (16B)│     │
│ 0x88 │ 含 SCI    │            │      │     │
│ E5   │ AN/PN/SL  │            │      │     │
└──────┴───────────┴────────────┴──────┴─────┘

- 逐跳加密：每台交换机独立加解密
- 保护整个原始帧（含原始 MAC 头）
- 用 MKA (MACsec Key Agreement) 协商密钥
- 典型应用：数据中心交换机互联、金融网络
```

#### WPA3 与 WPA2 对比
| 维度 | WPA2 | WPA3 |
|------|------|------|
| 握手 | 4-Way Handshake (PSK) | SAE (Simultaneous Authentication of Equals) |
| 抗字典攻击 | 弱（KRACK 攻击） | 强（前向保密 + 抗离线字典） |
| 加密算法 | CCMP (AES-128) | CCMP / GCMP-128 / GCMP-256 (WPA3-Enterprise) |
| OWE | 无 | Opportunistic Wireless Encryption（开放热点也加密） |

### L1 物理层加密

| 技术 | 场景 | 说明 |
|------|------|------|
| **光纤物理安全** | 骨干光缆 | 无电磁辐射，物理窃听需切断光纤（易检测） |
| **扩频/跳频 (DSSS/FHSS)** | 军事/蓝牙 | 信号分散在宽频带，抗干扰+低可检测 |
| **量子密钥分发 (QKD)** | 高安全场景 | 物理原理保证密钥安全，非计算复杂度 |
| **线路加密器 (inline encryptor)** | 政府/金融 | 物理线路两端硬件加密，线速 |

## IPsec VPN 完整工作流

```
阶段 1: IKE SA 建立 (UDP 500)
  ┌────────┐                            ┌────────┐
  │ 网关 A │── SA 提议 ────────────────→│ 网关 B │
  │        │←─ SA 选择 ──────────────── │        │
  │        │── KE (DH 公钥) + Nonce ───→│        │
  │        │←─ KE (DH 公钥) + Nonce ─── │        │
  │        │                            │        │
  │        │   双方各自: SK = DH(KE_A, KE_B)     │
  │        │   用 SK 认证 + 派生 IKE SA 密钥     │
  └────────┘                            └────────┘

阶段 2: IPsec SA 建立 (UDP 500, 加密)
  在加密的 IKE SA 通道内协商 IPsec SA:
    - 选择加密/认证算法
    - 选择模式 (传输/隧道)
    - 协商 SPI (安全参数索引)
    - 协商加密密钥

阶段 3: 数据传输
  [外层 IP 头] [ESP 头(SPI)] [加密的内层 IP+TCP+Data] [ESP 认证]
  ↑ 公网可见     ↑ 标识 SA    ↑ 完全加密                  ↑ HMAC

NAT 穿透 (NAT-T):
  IPsec ESP 包没有端口，NAT 设备难处理
  解决: 把 ESP 包封装在 UDP 4500 中传输
  [外层 IP][UDP:4500][ESP 包]
```

## 各层加密对比总览

| 层级 | 典型协议 | 保护范围 | 部署位置 | 性能 | 头部可见性 |
|------|----------|----------|----------|------|-----------|
| **L7** | PGP/Signal | 应用数据 | 端点应用 | 取决于应用 | 仅应用头部不可见 |
| **L4** | TLS 1.3 / QUIC | TCP/UDP 载荷 | 端点（进程） | 高（用户空间+内核） | L3/L2 头+端口可见（QUIC 端口也可见） |
| **L3** | IPsec ESP | IP 载荷（传输）/ 整包（隧道） | 主机或网关 | 最高（硬件加速） | 外层 IP 头可见 |
| **L2.5** | (无原生加密) | — | — | — | — |
| **L2** | MACsec / WPA3 | 整个 L2 帧 | 逐跳 | 线速 | 部分 L2 头可见 |
| **L1** | QKD / 线路加密器 | 比特流 | 物理线路 | 线速 | 不可见 |

## 常见组合场景

| 场景 | 加密组合 | 说明 |
|------|----------|------|
| **HTTPS 访问** | TLS 1.3 + (可选) MACsec | 端到端 TLS，数据中心内可叠加 L2 加密 |
| **企业站点 VPN** | IPsec 隧道模式 | 网关对网关，保护整个内网流量 |
| **远程办公 VPN** | WireGuard / OpenVPN / IPsec | 客户端到网关 |
| **Wi-Fi 上网** | WPA3 + TLS (访问 HTTPS) | 空口加密 + 端到端加密双层保护 |
| **数据中心 DCI** | MACsec + IPsec | L2 逐跳 + L3 跨站点端到端 |
| **邮件 E2E** | S/MIME 或 PGP + TLS | 内容端到端加密 + 传输层加密 |
| **运营商 MPLS VPN** | (无加密，可叠加 IPsec/MACsec) | 标签隔离，金融/政府客户通常要求加密 |

## 加密层级选择原则

| 需求 | 推荐层级 |
|------|----------|
| **端到端隐私（防中间任何人）** | L7 / L4（TLS、Signal） |
| **站点互联、全网加密** | L3（IPsec 隧道） |
| **单跳链路保护（防物理窃听）** | L2（MACsec） |
| **无线空口保护** | L2（WPA3） |
| **高安全等级（政府/军事）** | L1 + L3 叠加 |
| **性能优先 + 简单部署** | WireGuard (L3) |
| **现代 Web 应用标配** | L4 (TLS 1.3) |

> 注意：多层加密 ≠ 必然更安全。叠加时需注意：
> - 性能开销叠加
> - 内层加密被破坏不影响外层，但外层破坏可能暴露内层元数据
> - MTU 问题（每层加密都增加头部长度）
